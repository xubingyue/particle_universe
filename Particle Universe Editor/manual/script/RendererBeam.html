Beam Renderer

Renders a particle as a beam. The starting position of each beam is the position of the node to which the Particle System is attached. The end position is the particle position itself. The beam can be defined in such a way that it wiggles.

Definition

Syntax:

renderer   Beam   {renderer properties}

See ParticleRenderer for explanation of the syntax.

Example: 

technique
{
    renderer                                Beam
    {
        update_interval                     0.04
        beam_deviation                      1000
        number_of_segments                  1
    }
}

Properties

Use Vertex Colours

Set indication whether to use vertex colours or not. Syntax:

use_vertex_colours   true | false

use_vertex_colours is the keyword.
true | false If set to true, vertex colours are used. Default: false

Max Chain Elements

Maximum number of elements in a chain. The more elements, the smoother the beam. Syntax:

max_elements   #value

max_elements is the keyword.
#value Maximum of chain elements. Default: 10

Update Interval

The time it takes to change the wiggle of the beam. Syntax:

update_interval   #value

update_interval is the keyword.
#value defines when the beam changes the wiggle. Default: 0.1

Deviation

The deviation defines the amplitude of the wiggle. Syntax:

beam_deviation   #value

beam_deviation is the keyword.
#value defines the amplitude of the wiggle. Default: 300

Number of segments

The number of segments defines how many 'breakpoints' are used for the wiggle. The more segments, the more changes in the curve of the beam. The number of segments is not the same as the number of elements. Syntax:

number_of_segments   #value

number_of_segments is the keyword.
#value defines the curve of the wiggle. Default: 2

Jump

If jump is set to false, the wiggle of the beam is smooth. If set to true, the changes in the wiggle are 'jumpy'. Syntax:

beam_jump_segments   true | false

beam_jump_segments is the keyword.
true | false defines whether the wiggle jumps or not. Default: false

Texture Direction

Set the direction in which texture coords specified on each element are deemed to run along the length of the chain. Syntax:

beam_texcoord_direction   tcd_u | tcd_v

beam_texcoord_direction is the keyword.
tcd_u | tcd_v defines the direction. Default: tcd_v
