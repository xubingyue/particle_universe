Jet Affector

The Jet Affector changes the velocity of a particle. The difference between the Jet affector and the Linear Force affector is, that the Jet affector boosts the speed in a non-linear manner, while leaving the particle direction intact.

Definition

Syntax:

affector   Jet   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector               Jet
{
   acceleration        dyn_random
   {
      min              1
      max              10
   }
}

Properties


Acceleration

Sets the acceleration of the particle. Syntax:

acceleration
   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

acceleration is the keyword. jet_aff_accel is deprecated from version 1.2

#value is a real value that defines the acceleration, which increases (#value > 0.0) or decreases (#value < 0.0) the particle velocity. Default: 1.0.

dyn_random {random properties} means that the acceleration is generated randomly

dyn_curved_linear {control point [control point]} means that the acceleration is determined by means of 1 or more control points. The interpolation between the control points is linear. Note, that if this option is used, acceleration is based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_curved_spline {control point [control point]} means that the acceleration is determined by means of 1 or more control points. The interpolation between the control points is cubic. Note, that if this option is used, acceleration is based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_oscillate {oscillating properties} means that the acceleration is determined by means of an oscillating function.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.
