Particle Affector

Definition

The syntax of a Particle Affector definition is:

affector   [affectorName]   {affector properties}

affector is the keyword.
affectorType is the type of the affector.
affectorName defines the name of the affector. The affectorName is optional but required if the affector must be identified.
The affector properties are handled in the paragraph 'Base Properties'.

Example: 

affector               Gravity
{
    ...
}

Base Properties

The Base Properties apply to all affectors.

Enabled

Enables or disables the affector as soon as the particle system starts. Syntax:

enabled   true | false

enabled is the keyword.
true | false defines whether the affector is enabled (true) or disabled (false). Default: true

Position

Position applies to the relative position of this affector in respect to the Particle Technique. Syntax:

position   #x   #y   #z

position is the keyword.
#x is a real value that defines the x coordinate. Default: 0.
#y is a real value that defines the y coordinate. Default: 0.
#z is a real value that defines the z coordinate. Default: 0.

Mass

Defines the mass of a Particle Affector. This is only useful if the Particle Affector is also emitted and the mass is needed in another affector (so this affector is affected by another one). Although it is possible to define the mass of an affector particle in the Particle Emitter, it is also possible to define it here. Syntax:

mass_affector   #value

mass_affector is the keyword.
#value is a real value > 0 that defines the mass of the affector.

Exclude Emitter(s)

Exclude one or more Particle Emitters from this affector. This means that particles that are emitted by an emitter that is in the list of excluded emitters, will not be affected by this Particle Affector. Syntax:

exclude_emitter   #name   [#name]

exclude_emitter is the keyword.
#name   [#name] is an array of one or more emitter names that are excluded from the affector. Enclosing the names with double quotes is no longer supported from version 1.2

Affect Specialisation

A Particle Affector should sometimes behave different in various situations. These are common situations, which apply to most of the affectors. One such a common situation is that the affector strength is less when a particle is near the end of its lifetime. The Affect Specialisation is an optional property that affectors can use to differentiate behaviour in a number of common situations. If the options of this property apply to a particular affector, a 'logical' implementation has been choosen. Note, that this property is only implemented for a few affectors! Syntax:

affect_specialisation   special_default | special_ttl_increase | special_ttl_decrease

affect_specialisation is the keyword.
special_default is the default behaviour of an affector. It is not needed to set this property explicitly.
special_ttl_increase means that older particles will be affected MORE than just emitted particles.
special_ttl_decrease means that older particles will be affected LESS than just emitted particles.
