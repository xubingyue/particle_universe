Texture Rotator

The Texture Rotator rotates the texture of a particle.

Definition

Syntax:

affector   TextureRotator   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector                           TextureRotator
{
    use_own_rotation               true
    rotation                       dyn_random
    {
        min                        1
        max                        30
    }
    rotation_speed                 dyn_random
    {
        min                        1
        max                        30
    }
}

Properties


Use Own Rotation

This boolean attribute determines whether the rotation speed must be set and controlled by the particle affector, or whether the speed is set for each individual particle. In the first case, the rotation speed is the same for all particles and not set for individual particles. This also means that the rotation speed cannot be controlled by the particle itself (at least the Texture Rotator doesn't use it). In the second case, the rotation speed is set for each particle. Changing the rotation speed in the particle results in a different rotation speed of the texture. Syntax:

use_own_rotation   true | false

use_own_rotation is the keyword. tex_rot_use_own_rotation is deprecated from version 1.2
true | false is a boolean value that defines whether the rotation speed of the individual particle texture is set and used (true) or whether the rotation speed of the affector (the Texture Rotator) is used. Default: false.

Rotation

The rotation is a dynamic attribute that defines the initial rotation offset of a particle texture. Syntax:

rotation   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

rotation is the keyword. tex_rot_rotation is deprecated from version 1.2

#value is a real value that defines the initial texture rotation. Default: 0.0.

dyn_random {random properties} means that the initial texture rotation is generated randomly

dyn_curved_linear {control point [control point]} means that the initial texture rotation is determined by means of 1 or more control points. The interpolation between the control points is linear

dyn_curved_spline {control point [control point]} means that the initial texture rotation is determined by means of 1 or more control points. The interpolation between the control points is cubic

dyn_oscillate {oscillating properties} means that the initial texture rotation is determined by means of an oscillating function.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

Rotation Speed

The rotation speed is a dynamic attribute that defines the rotation speed of a particle texture. Syntax:

rotation_speed   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

rotation_speedis the keyword. tex_rot_speed is deprecated from version 1.2
#value is a real value that defines the rotation speed. Default: 0.0.

dyn_random {random properties} means that the rotation speed is generated randomly

dyn_curved_linear {control point [control point]} means that the rotation speed is determined by means of 1 or more control points. The interpolation between the control points is linear. Note, that if this option is used, speed is based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_curved_spline {control point [control point]} means that the rotation speed is determined by means of 1 or more control points. The interpolation between the control points is cubic. Note, that if this option is used, speed is based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_oscillate {oscillating properties} means that the rotation speed is determined by means of an oscillating function.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.
