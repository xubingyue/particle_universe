Scale Affector

The Scale Affector changes the size of a particle with a certain rate. The particle size can be enlarged or reduced.

Definition

Syntax:

affector   Scale   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector               Scale
{
   xyz_scale           10
}

Properties


Scale X, Y and Z

Sets the rate with which the particle size increases or decreases equally in all 3 directions. Syntax:

xyz_scale
   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

xyz_scale is the keyword.

#value is a real value that defines the rate with which the particle size increases (rate > 1.0) or decreases (rate < 1.0). All dimensions are scaled with the same value. Default: 1.0.

dyn_random {random properties} means that scaling is generated randomly

dyn_curved_linear {control point [control point]} means that scaling is determined by means of 1 or more control points. The interpolation between the control points is linear. Note, that if this option is used, scaling is done based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_curved_spline {control point [control point]} means that scaling is determined by means of 1 or more control points. The interpolation between the control points is cubic. Note, that if this option is used, scaling is done based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_oscillate {oscillating properties} means that scaling is determined by means of an oscillating function.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

Since Start System

Since Start System is an indication that determines whether scaling of the particles is determined from the start of the particle system or refers to the time fraction since the particle was emitted. Setting this property only has a meaning in cases where a curved (linear or spline) scaling type is used. If this property is set to true, the first value of each control point is the number of seconds from the start of the system. If the property is set to false, the first value of a control point is the fraction of the lifetime of a particle (lifetime particle / total lifetime). In the latter case, the first value of a control point lies in the range [0..1]. Syntax:

since_start_system   true | false

since_start_system is the keyword.
true | false refers to the time since the Particle System was started (true) or the time fraction of a particle. Default: false.
