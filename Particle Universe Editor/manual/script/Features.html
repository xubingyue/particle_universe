Features

Creation of particle systems by code or by script, which allows faster development of snow, rain, dust, fire, etc.

Complex scripting. For example, aliasses can be used to define a specific piece of reusable script to be used in other scripts.

More than 55 affectors, emitters, observers, handlers and renderers.

Advanced particle behaviour (flocking behaviour, particle collision).

Start, pause, resume and stop particle systems, which gives full control over the particle system lifecycle.

Multiple renderers within 1 particle system, allowing particle systems with different particle types and different materials.

Emission of visual particles, emitters, affectors and other particle systems.

Particle System LOD (= Level of Detail).

Use of functions (Random, Sine, Spline) instead of fixed value attributes only, for more control over a particle system.

PhysX™ support: Rigid body and fluid based particles.

Affectors are able to exclude certain emitters (particles emitted from these emitters will not be affected).

Soft particles: Particles intersecting other objects fade when they intersect objects, which doesn't result in sharp edges anymore.

Atlas texture support and AtlasImageTool for creating atlas textures.

Recording of a particle system to play it back as a video.

Particle Universe Serializer, which makes it possible to serialize a particle system to a script; for editor support.

Media pack with additional scripts and textures.
