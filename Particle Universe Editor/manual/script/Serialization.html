Serialization - writing to script

Deserialization - parsing a Particle Universe script - is a standard feature of the plugin and the manual covers the script options extensively. Serialization - creating a script from a Particle System - is an additional feature that is usually used in combination with an editor. Although the Particle Universe package doesn't contain an official editor yet, this serialization feature is one step in that direction. It is also usable for 3rd party editors and because it is part of the core, it is maintained and updated for every new version.

Usage

If you have created a Particle System by code, you can serialize it to a file by means of:

ParticleUniverse::ParticleSystemManager::getSingleton().writeScript(mSystem, "script_name.pu");

mSystem is a pointer to a created ParticleUniverse::ParticleSystem object.
"script_name.pu" is the name of the script file; "script_name.pu" is just an example.

It is possible to configure the serialization settings by means of the ParticleScriptSerializer. An example:

ParticleUniverse::ParticleScriptSerializer* serializer = 
    ParticleUniverse::ParticleSystemManager::getSingleton().getParticleScriptSerializer();
serializer->setPath(../../);

There is a second signature of the writeScript function. If the filename omits, the functions returns the script as a string:

Ogre::String scriptString = ParticleUniverse::ParticleSystemManager::getSingleton().writeScript(mSystem);
