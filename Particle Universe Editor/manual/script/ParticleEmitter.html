Particle Emitter

Definition

The syntax of a Particle Emitter definition is:

emitter   emitterType   [emitterName]   {emitter properties}

emitter is the keyword.
emitterType is the type of emitter, for example Point, Box or Line.
emitterName defines the name of the emitter. The emitterName is optional but required if the emitter must be identified. This is the case for example if the emitter must be emitted itself.
The emitter properties are handled in the paragraph 'Base Properties'.

Example: 

{
   ...
   emitter                                 Point
   {
       angle                               20
       emission_rate                       50
       direction                           0 1 0
       velocity                            dyn_random
       {
            min                            20
            max                            80
       }
       time_to_live                        dyn_random
       {
            min                            5
            max                            9
       }
   }
}

Base Properties

The Base Properties apply to all emitters.

Enabled

Enables or disables the emitter as soon as the particle system starts. Syntax:

enabled   true | false

enabled is the keyword.
true | false defines whether the emitter is enabled (true) or disabled (false). Default: true

Position

Position of the particle. The position is in local space and defines the relative position with the Particle Technique. Note, that this isn't the position of the emitter itself. Setting the emitter position isn't possible by means of a script. Syntax:

position   #x   #y   #z

position is the keyword.
#x is a real value that defines the x coordinate of the local space position vector. Default: 0.
#y is a real value that defines the y coordinate of the local space position vector. Default: 0.
#z is a real value that defines the z coordinate of the local space position vector. Default: 0.

Keep Local

If this attribute is set to 'true', the particles are moving relative to the emitter. Beware that this property can also be set in the particle system or the technique, which makes it possible to keep moving emitters, techniques, systems or visual particles local to respectively the particle system or the technique. Syntax:

keep_local   true | false

keep_local is the keyword.
true | false if set to 'true', the particles are moving relative to the emitter. Default: false.

Direction

Direction of the particle as soon as it is emitted. The direction is defined in local space in the script, but internally only world space is used. Syntax:

direction   #x   #y   #z

direction is the keyword.
#x is a real value that defines the x coordinate of the local space direction vector. Default: 0.
#y is a real value that defines the y coordinate of the local space direction vector. Default: 0.
#z is a real value that defines the z coordinate of the local space direction vector. Default: 0.

Orientation

The orientation of a particle that is emitted. Setting an orientation is only useful for 3D particles, such as Boxes and Spheres. Syntax:

orientation   #w  #x  #y  #z

orientation is the keyword.
#w  #x  #y  #z are the 4 values with which the orientation is identified. Internally these values are assigned to a Quaternion, which represents the orientation.

Start Orientation Range

With the use of Range Start and End it is possible to assign an orientation to a particle, which is randomly choosen 'between' a start and an end orientation. The Orientation Range Start represents one part of the interval. Syntax:

start_orientation_range   #w  #x  #y  #z

start_orientation_range is the keyword. range_start_orientation is deprecated from version 1.2
#w  #x  #y  #z are the 4 values with which the orientation is identified. Internally these values are assigned to a Quaternion, which represents the orientation.

End Orientation Range

With the use of Range Start and End it is possible to assign an orientation to a particle, which is randomly choosen 'between' a start and an end orientation. The Orientation Range End represents one part of the interval. Syntax:

end_orientation_range   #w  #x  #y  #z

end_orientation_range is the keyword. range_end_orientation is deprecated from version 1.2
#w  #x  #y  #z are the 4 values with which the orientation is identified. Internally these values are assigned to a Quaternion, which represents the orientation.

Velocity

Defines the initial velocity of a particle as soon as it is emitted. This attributes distinguishes 5 appearance types. Syntax:

velocity
   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

velocity is the keyword.

#value is a fixed (real) value > 0 that defines the velocity. Default: 1.

Example: 

velocity                100


dyn_random {random properties} means that the velocity is randomly generated. Syntax random properties:
   min   #minValue defines the lower bound of the random number.
   max   #maxValue defines the upper bound of the random number.

Example: 

velocity                dyn_random
{   
    min                 20
    max                 80
}


dyn_curved_linear {control point [control point]} means that the velocity is determined by means of 1 or more control points. The interpolation between the control points is linear. Syntax control points:
   control_point   #time   #value.
   #time is the time since the Particle System was started.
   #value is a (real) value that defines the velocity on that time.

Example: 

velocity                dyn_curved_linear
{
    control_point       0 300
    control_point       1 200
    control_point       3 500
    control_point       4 10
}

Note, that control points don't have to be added in a particular order.


dyn_curved_spline {control point [control point]} means that the velocity is determined by means of 1 or more control points. The interpolation between the control points is cubic. Syntax see dyn_curved_linear:

Example: 

velocity                dyn_curved_spline
{
    control_point       0 300
    control_point       1 200
    control_point       3 500
    control_point       4 10
}


dyn_oscillate {oscillating properties} means that the velocity is determined by means of an oscillating function (sine, for example).

Example: 

velocity                 dyn_oscillate
{
    oscillate_frequency  0.5
    oscillate_base       100
    oscillate_amplitude  60
    oscillate_type       sine
}

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

repeat_delay is the keyword.
See the 'velocity' property for explanation of the 5 property appearances.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

Angle

Defines the deviation of the particles' direction. Syntax:

angle
   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

angle is the keyword.
See the 'velocity' property for explanation of the 5 property appearances.
Remark: If the angle is represented by an #value, that value lies in the interval 0 - 360. If the angle is set to a fixed value of 40, the deviation of the particles' direction is (random) between 0 and 40 degrees. Default: 0

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

Emission Rate

The emission rates defines how many particles are emitted in a second. Syntax:

emission_rate
   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

emission_rate is the keyword.
See the 'velocity' property for explanation of the 5 property appearances. See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

Note, that emission rate is an attribute that can be influenced by a camera dependency using the << camera_dependency construction.

Texture Coordinates

Set the initial texture coordinates of a particle. This is only useful if multiple texture coords are defined in the renderer (by means of texture_coords_rows and texture_coords_columns). Syntax:

texture_coords   #value

texture_coords is the keyword.
#value defines the texture coordinates. Default: 0

Start Texture Coordinates Range

If no fixed texture coordinates are set, it is also possible to generate a random texture coordinate between a 'start' and an 'end'. This is only useful if multiple texture coords are defined in the renderer. Syntax:

start_texture_coords_range   #value

start_texture_coords_range is the keyword. start_texture_coords is deprecated from version 1.2
#value defines the lower texture coordinate. Default: 0

End Texture Coordinates Range

If no fixed texture coordinates are set, it is also possible to generate a random texture coordinate between a 'start' and an 'end'. This is only useful if multiple texture coords are defined in the renderer. Syntax:

end_texture_coords_range   #value

end_texture_coords_range is the keyword. end_texture_coords is deprecated from version 1.2
#value defines the upper texture coordinate. Default: 0

Colour

Set the initial colour of a particle. Syntax:

colour   #red   #green   #blue   [#alpha]

colour is the keyword.
#red defines the red value (between 0..1).
#green defines the green value (between 0..1).
#blue defines the blue value (between 0..1).
#alpha is an optional value and only added when needed (for example if scene_blend alpha_blend is used in the material). Its value lies between 0..1.
Default colour: White

Start Colour Range

If no fixed colour is set, it is also possible to generate a random colour between a 'start' colour and an 'end' colour. Syntax:

start_colour_range   #red   #green   #blue   [#alpha]

start_colour_range is the keyword.
#red defines the red value (between 0..1).
#green defines the green value (between 0..1).
#blue defines the blue value (between 0..1).
#alpha is an optional value and only added when needed (for example if scene_blend alpha_blend is used in the material). Its value lies between 0..1.
Default colour: Black

End Colour Range

If no fixed colour is set, it is also possible to generate a random colour between a 'start' colour and an 'end' colour. Syntax:

end_colour_range   #red   #green   #blue   [#alpha]

end_colour_range is the keyword.
#red defines the red value (between 0..1).
#green defines the green value (between 0..1).
#blue defines the blue value (between 0..1).
#alpha is an optional value and only added when needed (for example if scene_blend alpha_blend is used in the material). Its value lies between 0..1.

All Particle Dimensions

   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

all_particle_dimensions is the keyword.
See the 'velocity' property for explanation of the 5 property appearances.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

Auto Direction

Besides setting the particle direction manually, it is also possible to let the system generate the particle direction. This is often done for emitters with a particular shape and not used in all emitters. If this attribute is true, the direction vector that has been set, is ignored. Syntax:

auto_direction   true | false

auto_direction is the keyword.
true | false determines whether auto direction is used (true) or not. Default: false.

Force Emission

The emission rate defines the number of particles that are emitted over a period of time. In some cases however, you would like to emit the particle all at once, without any delay. That is the purpose of this attribute. If the Force Emission is set to true, all particles defined in Emission Rate are emitted at once. The emitter stops emitting particles after that. Syntax:

force_emission   true | false

force_emission is the keyword.
true | false determines whether all particles defined in emission_rate are emitted at once (true) or not. Default: false.

Emits

Defines the type of emitted particle. The use is very versatile. Besides emitting components that are registered within the same Particle System, also other Particle Systems can be emitted.

A word of caution: If you are emitting components that include emitters, which on their turn emit other components, the number of pre-allocated objects can become very large. Use the ..._quota properties in the techniques to manage it. Syntax:

emits   visual_particle | emitter_particle | technique_particle | affector_particle | system_particle   #name

emits is the keyword.
visual_particle | emitter_particle | technique_particle | affector_particle | system_particle One of these 5 literals must used to identify which type of particle is emitted. Default: visual_particle.
#name In case no visual particle is emitted, the name (name in the script, not the runtime name) of the particle type must be supplied. So, if the particle type is an emitter, the name of that emitter is needed.

Example:

// This emitter emits other emitters with the name 'Splash'
emitter               Point
{
    ...
    emits             emitter_particle      Splash
}


// This is the named emitter that is emitted
emitter               Point                 Splash
{
    ...
}
