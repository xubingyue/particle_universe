PhysX Fluid Extern

Nvidia's PhysX can be used to add physical behaviour to Particle Universe particles. Currently there are 2 options:
1) Rigid body based particles: Particle Universe particles are associated with an NxActor. This means that particles derive their values (position, direction, ...) from the NxActor. This is the default behaviour of a PhysX Extern.
2) Fluid based particles: Particle Universe particles are associated with an NxFluid.

Definition

Each particle system that uses the PhysFluid capabilities must add a PhysXFluid Extern to its technique. With the use of the PhysXFluid Extern, the PhysX properties are assigned to the fluid particles. Syntax:

extern   PhysXFluid   [externName]   {extern properties}

See Extern for explanation of the syntax.

Script example: 

technique
{
    ....
    extern                                  PhysXFluid
    {
        rest_particles_per_meter            0.2
        rest_density                        20
        kernel_radius_multiplier            1.9
        motion_limit_multiplier             2
        collision_distance_multiplier       1.6
        packet_size_multiplier              4
        stiffness                           0.1
        viscosity                           0.1
        surface_tension                     0.1
        damping                             1
        external_acceleration               0 -5 0
        collision_group                     2
        flags
        {
            hardware                        false
        }
    }
}

Properties


Rest Particle Per Meter

The number of particles per meter in the relaxed state of the fluid. Syntax:

rest_particles_per_meter   #value

rest_particles_per_meter is the keyword.
#value is a floating value and defines the rest particle per meter.

Rest Density

The density in the relaxed state of the fluid. Syntax:

rest_density   #value

rest_density is the keyword.
#value is a floating value and defines the rest density.

Kernel Radius Multiplier

The particle interacts with a certain radius. Syntax:

kernel_radius_multiplier   #value

kernel_radius_multiplier is the keyword.
#value is a floating value and defines the kernel radius multiplier.

Motion Limit Multiplier

The particle can move a maximal distance of during one timestep. Syntax:

motion_limit_multiplier   #value

motion_limit_multiplier is the keyword.
#value is a floating value and defines the motion limit multiplier.

Collision Distance Multiplier

The distance between particles and collision geometry, which is maintained during simulation. Syntax:

collision_distance_multiplier   #value

collision_distance_multiplier is the keyword.
#value is a floating point value and defines the collision distance multiplier.

Packet Size multiplier

The fluid packet size used for parallelization of the fluid computations. Syntax:

packet_size_multiplier   #value

packet_size_multiplier is the keyword.
#value is an unsigned int (32 bits) value and defines the packet size multiplier.

Stiffness

Set the fluid stiffness. Syntax:

stiffness   #value

stiffness is the keyword.
#value is a floating point value and defines the stiffness.

Viscosity

Set the fluid viscosity. Syntax:

viscosity   #value

viscosity is the keyword.
#value is a floating point value and defines the viscosity.

Surface Tension

Set the fluid surface tension. Syntax:

surface_tension   #value

surface_tension is the keyword.
#value is a floating point value and defines the surface tension.

Damping

Set the fluid damping. Syntax:

damping   #value

damping is the keyword.
#value is a floating point value and defines the damping.

External Acceleration

Set the external acceleration applied to each particle at each time step. Syntax:

external_acceleration   #x #y #z

external_acceleration is the keyword.
#x #y #z is a vector3 and defines the external acceleration.

Restitution For Static Shapes

Set the restitution used for collision with static shapes. Syntax:

restitution_for_static_shapes   #value

restitution_for_static_shapes is the keyword.
#value is a floating point value and defines the restitution for static shapes.

Dynamic Friction For Static Shapes

The dynamic friction used for collision with static shapes. Syntax:

dynamic_friction_for_static_shapes   #value

dynamic_friction_for_static_shapes is the keyword.
#value is a floating point value and defines the dynamic friction for static shapes.

Static Friction For Static Shapes

Currently not implemented. Syntax:

static_friction_for_static_shapes   #value

static_friction_for_static_shapes is the keyword.
#value is a floating point value and defines the static friction for static shapes.

Attraction For Static Shapes

Currently not implemented. Syntax:

attraction_for_static_shapes   #value

attraction_for_static_shapes is the keyword.
#value is a floating point value and defines the attraction for static shapes.

Restitution For Dynamic Shapes

The restitution used for collision with static shapes. Syntax:

restitution_for_dynamic_shapes   #value

restitution_for_dynamic_shapes is the keyword.
#value is a floating point value and defines the restitution for dynamic shapes.

Dynamic Friction For Dynamic Shapes

The dynamic friction used for collision with dynamic shapes. Syntax:

dynamic_friction_for_dynamic_shapes   #value

dynamic_friction_for_dynamic_shapes is the keyword.
#value is a floating point value and defines the dynamic friction for dynamic shapes.

Static Friction For Dynamic Shapes

Currently not implemented. Syntax:

static_friction_for_dynamic_shapes   #value

static_friction_for_dynamic_shapes is the keyword.
#value is a floating point value and defines the static friction for dynamic shapes.

Attraction For Dynamic Shapes

Currently not implemented. Syntax:

attraction_for_dynamic_shapes   #value

attraction_for_dynamic_shapes is the keyword.
#value is a floating point value and defines the attraction for dynamic shapes.

Collision Response Coefficient

Set the collision response coefficient. Syntax:

collision_response_coefficient   #value

collision_response_coefficient is the keyword.
#value is a floating point value and defines the collision response coefficient.

Collision Group

Defines which collision group this fluid is part of. Syntax:

collision_group   #value

collision_group is the keyword.
#value is an unsigned integer value and defines the collision group.

Simulation Method

Describes the particle simulation method. Particles can be treated in two ways: either they are simulated considering interparticular forces, or they are simulated independently. In the latter case, you still get collision between particles and static/dynamic shapes, damping, acceleration due to gravity, and the user force. Syntax:

simulation_method   intercollision | no_intercollision | mix_intercollision

simulation_method is the keyword.
intercollision | no_intercollision | mix_intercollision is the simulation method. Default: intercollision

Collision Method

The collision method of the fluid. Syntax:

collision_method   static | dynamic

collision_method is the keyword.
static | dynamic is the method of collision. Default: static

Flag - Visualization

Enables debug visualization for the fluid. Syntax:

visualization   true | false

visualization is the keyword.
true | false defines whether visualisation is set.

Flag - Disable Gravity

Disables scene gravity for the fluid. Syntax:

disable_gravity   true | false

disable_gravity is the keyword.
true | false defines whether gravity is disabled.

Flag - Collision Two Way

Enable/disable two way collision of fluid with the rigid body scene. In either case, fluid is influenced by colliding rigid bodies. If this flag is not set, rigid bodies are not influenced by colliding pieces of fluid. Syntax:

collision_twoway   true | false

collision_twoway is the keyword.
true | false defines whether two way collision is used.

Flag - Fluid Enabled

Enable/disable execution of fluid simulation. Syntax:

fluid_enabled   true | false

fluid_enabled is the keyword.
true | false defines whether the fluid is enabled.

Flag - Hardware

Defines whether this fluid is simulated on the PPU. Syntax:

hardware   true | false

hardware is the keyword.
true | false defines whether a hardware (PPU) is used. Default: false

Flag - Priority Mode

Enable/disable particle priority mode. If enabled, the oldest particles are deleted to keep a certain budget for new particles. Syntax:

priority_mode   true | false

priority_mode is the keyword.
true | false defines whether the priority mode is set.

Flag - Project To Plane

Defines whether the particles of this fluid should be projected to a plane. This can be used to build 2D fluid applications, for instance. Syntax:

project_to_plane   true | false

project_to_plane is the keyword.
true | false defines whether the particles are projected to a plane.

Flag - Strict Cooking Format

Currently not implemented. Syntax:

strict_cooking_format   true | false

strict_cooking_format is the keyword.
true | false defines whether strict cooking format is used.

Group Mask

128-bit mask used for collision filtering. Syntax:

group_mask   #bits0   #bits1   #bits2   #bits3

group_mask is the keyword.
#bits0   #bits1   #bits2   #bits3 represent the four 32 bit values.
