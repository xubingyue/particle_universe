Velocity Matching Affector

This affector is one element of the Flocking behaviour algorithm. It adjusts the velocity of a particle so it matches the velocity of its neighbouring particles. The method of detecting whether particles are nearby is not done by a brute-force method, but with a spatial hashing algorithm. This means that in the Particle Technique, you have to set the spatial hashing properties (i.e. 'spatial_hashing_cell_dimension').

Definition

Syntax:

affector   VelocityMatching   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector                        VelocityMatching
{
   radius                       100
}

Properties


Radius

This is the maximum distance between 2 particles before the velocity matching function comes into action. This means that if the positions of two particles < Radius, the velocities of both particles are adjusted. Note, that it is called radius, because it is comparable with a bounding sphere collision detection. Syntax:

radius   #value

radius is the keyword. velocity_matching_radius is deprecated from version 1.2
#value is the maximum distance between 2 particles, before the velocities are adjusted. Default: 100.
