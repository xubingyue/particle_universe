Scale Velocity Affector

The Scale Velocity Affector changes the velocity of a particle with a certain rate, without affecting the direction.

Definition

Syntax:

affector   ScaleVelocity   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector                  ScaleVelocity
{
    velocity_scale        dyn_curved_linear
    {
        control_point     0 -1800
        control_point     0.14 -50
    }
}

Properties


Velocity Scale

Sets the rate with which the particle velocity decreases (or increases). Syntax:

velocity_scale
   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

velocity_scale is the keyword.

#value is a real value that defines the rate with which the particle velocity increases (> 1.0) or decreases (< 1.0). Default: 1.0.

dyn_random {random properties} means that velocity scaling is generated randomly

dyn_curved_linear {control point [control point]} means that velocity scaling is determined by means of 1 or more control points. The interpolation between the control points is linear. Note, that if this option is used, velocity scaling is done based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_curved_spline {control point [control point]} means that velocity scaling is determined by means of 1 or more control points. The interpolation between the control points is cubic. Note, that if this option is used, velocity scaling is done based on the particles' time-to-live and not based on the time since the start of the particle system. This means the the first value of each control point lies between the interval [0..1].

dyn_oscillate {oscillating properties} means that velocity scaling is determined by means of an oscillating function.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.

Since Start System

Since Start System is an indication that determines whether velocity scaling is determined from the start of the particle system or refers to the time fraction since the particle was emitted. Setting this property only has a meaning in cases where a curved (linear or spline) scaling type is used. If this property is set to true, the first value of each control point is the number of seconds from the start of the system. If the property is set to false, the first value of a control point is the fraction of the lifetime of a particle (lifetime particle / total lifetime). In the latter case, the first value of a control point lies in the range [0..1]. Syntax:

since_start_system   true | false

since_start_system is the keyword.
true | false refers to the time since the Particle System was started (true) or the time fraction of a particle. Default: false.

Stop At Flip

When the velocity increases or decreases, the direction is influenced in case the velocity flips from a positive value to a negative value or vice versa. For example, if the direction is set to (1, 0.5, 0) and the velocity is decreased, at some point, the velocity becomes zero. If the affector continues to decrease the velocity, the direction is reversed to (-1, -0.5, 0) and the particle goes the opposite direction. To prevent this, the 'Stop At Flip' value can be set to 'true'. Syntax:

stop_at_flip   true | false

stop_at_flip is the keyword.
true | false defines whether the affector continues to increase/decrease, when the velocity 'flips' from positive to negative or vice versa. Default: false.
