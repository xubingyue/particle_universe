Position Emitter

Emits particles at one or more positions, which are added to this emitter. This emitter is usable to create a structure (a 'wall' for example) of particles, where the emitter contains all positions of that structure. Adding physical behaviour to the particles enables the structure to behave naturally (i.e. collapses after impact).

Definition

Syntax:

emitter   Position   [emitterName]   {emitter properties}

See ParticleEmitter for explanation of the syntax

Example: 

emitter                Position
{
    emission_rate      10
    force_emission     true
    time_to_live       100
    random_position    false
    add_position       200 100 300
    add_position       250 100 300
    add_position       300 100 300
    add_position       350 100 300
    add_position       400 100 300
}

Properties


Add Position

Positions can be added to this emitter and particles are emitted from these positions. DonÂ´t confuse this property with the 'position' property. The 'position' property sets the position of the emitter as a whole (relative to its parent), while the 'add_position' property defines positions relative to the emitter. Syntax:

add_position   #x #y #z

add_position is the keyword.
#x is the x-value of the position.
#y is the y-value of the position.
#z is the z-value of the position.

Random Position

Particles are emitted at one of the positions that are added to this emitter. The position can be picked random from the list, or the positions are picked in the order they where added. Note, that if the emission_rate value is larger than the number of added positions and force_emission has been set to true, the number of emitted particles is not determined by the emission_rate value, but by the number of added positions. Syntax:

random_position   true | false

random_position is the keyword.
true | false defines whether the particles must be emitted random or not. Default: true.
