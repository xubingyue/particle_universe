Do Affector Event Handler

Forces an affector to affect the particle. This might look odd, because if a Particle Technique contains the Particle Affector, the particles are already affected. The reason why this Event Handler exists is that in some situations for example, you only want to affect the particle once (at a certain event). Using this Event Handler has a few restrictions:
- the _preProcessParticles() and the _postProcessParticles() of the Particle Affector are not called by default; only the _affect() function is called. However, you are able to set an indication to perform the _preProcessParticles() and _postProcessParticles, but this is done per particle!
- the Particle Affector must still be included in the Particle Technique. If the Particle Affector is enabled, an additional call to the affector is executed when a certain event occurs. If the Particle Affector is disabled, the 'affect' is executed only once when a certain event occurs.

Definition

Syntax:

handler   DoAffector   [handlerName]   {handler properties}

See ParticleEventHandler for explanation of the syntax.

Example: 

observer                     OnExpire
{
   handler                   DoAffector
   {
      force_affector         myAffector
   }
}

Properties


Pre/Post Processing

Indication to determine that the_preProcessParticles() and the _postProcessParticles() must also be executed. Syntax:

pre_post   true | false

pre_post is the keyword.
true | false is a boolean value to indicate that the_preProcessParticles() and the _postProcessParticles() must also be executed. Default: false

Force Affector

This defines the name of the affector that is forced. The affector will be searched in the technique that also contains this event handler. If the affector cannot be found, it will search in the other techniques. Syntax:

force_affector   #affectorName

force_affector is the keyword.
#affectorName is the name of the affector.
