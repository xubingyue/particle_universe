Do Placement Particle Event Handler

Force the emission of one or more new particles. The position at which the particles are emitted is defined by the particle that has been observed by the observer at which this event handler is registered. One application where you can use the Placement Particle Event Handler is to put a splash on the position where a raindrop hits the ground.

Definition

Syntax:

handler   DoPlacementParticle   [handlerName]   {handler properties}

See ParticleEventHandler for explanation of the syntax.

Example: 

handler                   DoPlacementParticle
{
   force_emitter          myEmitter
   number_of_particles    1
}

Properties


Number of Particles

The number of particles that are definately emitted. Syntax:

number_of_particles   #value

number_of_particles is the keyword.
#value defines the number of particles to be emitted.

Force Emitter

This property sets the name of the emitter that is forced to emit one or more particles. Syntax:

force_emitter   #emitterName

force_emitter is the keyword.
#emitterName is the name of the emitter that is forced to emit one or more particles.

Inherit Position

Defines whether the placed particle initially (when emitted) gets the same position as the particle for which an event occurs (particle expires, freezes, emits). Syntax:

inherit_position   true | false

inherit_position is the keyword.
true | false determines whether the placed particle gets the same position during emission. Default: true
