Slave Emitter

The Slave Emitter is no real emitter in a strict sense. As the name suggests, it acts as a slave. The Master is another emitter, usually from another technique. Its use? Well, the Slave Emitter places a particle on the same position (and with the same direction) as the 'master' particle. Using the right materials and renderers it is possible to create complex effects. To expand the possibilities, you can use it in combination with a SlaveBehaviour. If both a Slave Emitter and Slave Behaviour are included in the technique, the emitted particle will not only have the same initial position and direction of its master-particle, but also continues to follow it, regarding the movement of the master-particle.

Definition

Syntax:

emitter   Slave   [emitterName]   {emitter properties}

See ParticleEmitter for explanation of the syntax

Example: 

emitter                      Slave
{
   ...
   master_technique_name     flareTechnique
   master_emitter_name       flareEmitter
}

Properties


Master Technique Name

The Slave Emitter emits a particle as soon as a particle is emitted from an emitter in the technique that is specified with this name. Syntax:

master_technique_name   #name

master_technique_name is the keyword.
#name is the name of the master technique.

Master Emitter Name

If also the master emitter name is set, you can narrow the particles that act as a master. Only particles that have been emitted from the emitter with this name are followed. Note, that the name of the master technique is mandatory, while the name of the master emitter is optional. Syntax:

master_emitter_name   #name

master_emitter_name is the keyword.
#name is the name of the master emitter.
