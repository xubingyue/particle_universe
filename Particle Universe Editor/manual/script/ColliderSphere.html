Sphere Collider

The Sphere Collider represents a sphere shape with which the particles collide. This concerns particles of the same technique at which the Sphere Collider is registered. If a particle collides with the shape, the collision flag is set, which makes it possible to observe the particle collision. Furthermore, the Collider itself can change the behaviour of the particle. This concerns behaviour that is difficult to realise (or with a lot of overhead) in an event handler, because the data will not be available in the event handler. Bouncing and flowing (in which the particle flows on the surface of the shape) are typical examples.

Beware that the Sphere Collider is part of Particle Universe itself. If you want more sophisticated collision and responses, Particle Universe also has the option to integrate with NvidiaÂ´s PhysX engine, formerly owned by Ageia.

Definition

Syntax:

affector   SphereCollider   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector                     SphereCollider
{
    position                 0 400 0
    radius                   300
    intersection             point
    collision_type           bounce
    friction                 0.2
    bouncyness               0.4
}

Properties


Friction

Friction is an attribute of the sphere surface. It indicates the smoothness of the surface, which will influence the rotation behaviour of the particle when it hits the sphere. If the friction is high, the particle rotation (both texture and geometry rotation) decreases. Syntax:

friction   #value

friction is the keyword. collision_friction is deprecated from version 1.2
#value is a real value > 0 that defines the friction. Default: 0.0.

Bouncyness

Bouncyness is an attribute of the surface. It indicates how much a particle bounces from the sphere when it hits the surface. This means that bouncyness influences the particle velocity. Syntax:

bouncyness   #value

bouncyness is the keyword. collision_bouncyness is deprecated from version 1.2
#value is a real value between > 0 that defines the bouncyness. The higher the value, the higher the velocity of a particle when it hits the sphere. Default: 1.0.

Intersection Type

Determines how a particle collision should be determined. 'point' means that the position of a particle will be validated against the Colliders' shape, while 'box' means that the dimensions (width, height and depth) are taken into account to determine whether the particle collides. Syntax:

intersection   point | box

intersection is the keyword. collision_intersection is deprecated from version 1.2
point defines whether the position of the particle is used to determine collision. Default: point.
box defines whether the particle width, height and depth are used to determine collision.

Collision Type

Determines how a particle behaves after collision with this collider. The behaviour of the particle is solved in the collider and only behaviour that needs the colliders' data, is taken into account. The fact that a particle expires for example, can be achieved by using an Observer in combination with an EventHandler (DoExpireEventHandler). Syntax:

collision_type   bounce | flow | none

collision_type is the keyword.
bounce means that the particle bounces off the collider. Default: bounce.
flow means that the particle flows around the contours of the collider.
none means that the particle doesn't do anything. This value should be set if the behaviour of the particle is determined outside the collider (for example, expiring the particle).

Radius

Represents the radius of the sphere. Syntax:

radius   #value

radius is the keyword. sphere_collider_radius is deprecated from version 1.2
#value is a real value > 0 that defines the radius of the sphere. Default: 0.0.

Inner Collision

Defines whether the collision takes place inside of the sphere, or outside. If the value is set to true, particles collide with the inner surface of the sphere. Syntax:

inner_collision   true | false

inner_collision is the keyword.
true | false defines wither the collision is inside of the sphere. Default: false.
