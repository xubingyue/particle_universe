On Event Flag Observer

Some of the Observers (OnEmission, OnCollision) validate the value of the particles' event flag. These observers are created just for convenience. A more generic solution is this On Event Flag Observer. You can set a mask (the event_flag) in the script in the form of a uint32. With this mask it is possible to validate more than one particle state at once. For instance, if you want to determine whether a particle has collided and expired at the same time, the event flag must be set to 1 (expired) + 4 (collided) = 5.

Instead of using predefined values, it is also possible to validate the flag with a user value. It is possible to add or set any event flag to a particle, which can be detected by this observer.

Definition

Syntax:

observer   OnEventFlag   [observerName]   {observer properties}

See ParticleObserver for explanation of the syntax.

Example: 

observer                OnEventFlag
{
   event_flag           4
   handler              DoExpire {}
}

Properties


Event Flag

The event flag is a mask that is used to validate the event flag value of a particle. The mask and the particles' event flag value are matched bitwise. If a bit in the mask has been set in the event flag of the particle, the observer calls the registered handlers. Syntax:

event_flag   #value

event_flag is the keyword.
#value is a uint32 value that is used for matching the eventflag of the particle. Default: 0.
