Particle Follower

The Particle Follower forces a particle to follow its predecessor if it is within a certain range of the previous particle. This affector makes it possible to create a trail of particles

Definition

Syntax:

affector   Follow   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector                           Follower
{
    min_distance                   60
    max_distance                   300
}

Properties


Minimum Distance

If the particle is beyond the minimum distance of its predecessor, it will be forced to follow its predecessor within the limits defined by the minimum distance. Syntax:

min_distance   #value

min_distance is the keyword. follower_min_distance is deprecated from version 1.2
#value is a real value that defines the minimum between 2 successive particles. Default: 10.0.

Maximum Distance

If the maximum distance has been defined, and a particle is beyond this distance (in comparison with the previous particle), the 'follow' effect is ignored. In some cases this can be convenient. If, for example the emission rate is low and the time to live is high, new emitted particles always tend to stick at the back of the particle 'trail'. If the maximum distance is set, a new emitted particle ignores this trail if it is too far away and starts a new trail. Syntax:

max_distance   #value

max_distance is the keyword. follower_max_distance is deprecated from version 1.2
#value is a real value that defines the maximum distance between 2 successive particles for which the follow effect is ignored. Default: Infinite.
