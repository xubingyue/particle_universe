Billboard Renderer

Renders particles as a billboard, using the material set in the technique.

Definition

Syntax:

renderer   Billboard   {renderer properties}

See ParticleRenderer for explanation of the syntax.

Example: 

renderer                Billboard
{
   billboard_type       point
   sorting              true
}

Properties


Billboard Type

Set the type of billboard. The different types also influence the way (orientation) the billboard are rendered. Syntax:

billboard_type   point | oriented_common | oriented_self | perpendicular_common | perpendicular_self | oriented_shape

billboard_type is the keyword.
point | oriented_common | oriented_self | perpendicular_common | perpendicular_self | oriented_shape. These types are the same as the ones defined in Ogre's ParticleFX plugin, except for oriented_shape, which uses the orientation of the particle to change the direction. Default: point

Origin

Set the relative offset of the billboard in relation to its position. Syntax:

billboard_origin   top_left | top_center | top_right | center_left | center_right | center | bottom_left | bottom_center | bottom_right

billboard_origin is the keyword.
top_left | top_center | top_right | center_left | center_right | center | bottom_left | bottom_center | bottom_right. These origins are the same as the ones defined in Ogre's ParticleFX plugin. Default: center

Rotation Type

It is possible to rotate the billboard around the direction it faces. This is achieved by either rotating the vertices or the texture coorcdinates. Syntax:

billboard_rotation_type   vertex | texcoord

billboard_rotation_type is the keyword.
vertex | texcoord. These rotation types are the same as the ones defined in Ogre's ParticleFX plugin.Default: texcoord

Common Direction

If the billboard_type is set to oriented_common or perpendicular_common, the direction vector is identified by this attribute. Syntax:

common_direction   #x #y #z

common_direction is the keyword.
#x represents the x-coordinate of the common direction vector. Default: 0
#y represents the y-coordinate of the common direction vector. Default: 0
#z represents the z-coordinate of the common direction vector. Default: 0

(Common) Up Vector

If the billboard_type is set to either perpendicular_common or perpendicular_self, the up vector is required to determine the billboard x- and y-axis. Syntax:

common_up_vector   #x #y #z

common_up_vector is the keyword.
#x represents the x-coordinate of the common up vector. Default: 0
#y represents the y-coordinate of the common up vector. Default: 0
#z represents the z-coordinate of the common up vector. Default: 0

Point Rendering

Use hardware point-rendering instead of using textured quads. Syntax:

point_rendering   true | false

point_rendering is the keyword.
true | false. Determines whether the billboard must be rendered as a point (true) or not. Default: false

Accurate Facing

Using a billboard facing model, which is more accurate than the default facing model. Syntax:

accurate_facing   on | off

accurate_facing is the keyword.
on | off. Determines whether accurate facing is 'on' or 'off'. Default: off
