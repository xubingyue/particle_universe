Special Cases

The scripting possibilities of the Particle Universe plugin allows complex constructions. Because some of these constructions show up often, they are listed in separate paragraphs.

Dynamic Attribute

A dynamic attribute is an attribute that can be defined in multiple ways. Besides assigning a fixed value to the attribute, it is also possible to generate a random value or picking a value on a curve. The dynamic attribute has 5 possible appearance types.

Syntax 1

Defines a fixed value.

attribute   #value

attribute is the keyword of the attribute.
#value is a fixed (real) value.

Example: 

attribute               100
Syntax 2

Defines a random value.

attribute   dyn_random   {#minValue   #maxValue}

attribute is the keyword of the attribute.
dyn_random means that the value is randomly generated.
#minValue defines the lower bound of the random number.
#maxValue defines the upper bound of the random number.

Example: 

attribute               dyn_random
{   
    min                 20
    max                 80
}
Syntax 3

The value is linear interpolated on a curve that is defined by means of 1 or more control points.

attribute   dyn_curved_linear   {control_point   #time   #value   [control_point   #time   #value]}
attribute is the keyword of the attribute.
dyn_curved_linear The interpolation between the control points is linear.
control_point is the keyword for a control point.
#time is the time since the Particle System was started.
#value defines the value on that time.

Example: 

attribute               dyn_curved_linear
{
    control_point       0 300
    control_point       1 200
    control_point       3 500
    control_point       4 10
}

Note, that control points don't have to be listed in a particular order.

Syntax 4

The value is cubic interpolated on a curve that is defined by means of 1 or more control points.

attribute   dyn_curved_spline   {control_point   #time   #value   [control_point   #time   #value]}
attribute is the keyword of the attribute.
dyn_curved_spline The interpolation between the control points is cubic.
control_point is the keyword for a control point.
#time is the time since the Particle System was started.
#value defines the value on that time.

Example: 

attribute               dyn_curved_spline
{
    control_point       0 300
    control_point       1 200
    control_point       3 500
    control_point       4 10
}
Syntax 5

The value is oscillating. Example: A sine function.

attribute   dyn_oscillate   {#oscillate_frequency   #oscillate_phase   #oscillate_base
#oscillate_amplitude   #oscillate_type}

attribute is the keyword of the attribute.
dyn_oscillate The value is generated by means of an oscillating function.
#oscillate_frequency is the frequency of the oscillating function (cycles per second).
#oscillate_phase is phase of the oscillating function.
#oscillate_base is a base value that is added to the result. For example, if the oscillating function returns values between -10 and 10, an #oscillate_base value of 100 results in returning values between 90 and 110.
#oscillate_amplitude is the amplitude of the oscilating function.
#oscillate_type is sine | square.
Example: 

emission_rate              dyn_oscillate
{
    oscillate_frequency    0.5
    oscillate_phase        0
    oscillate_base         100
    oscillate_amplitude    40
    oscillate_type         sine
}

Camera Dependency

A dependency defines a relationship between 2 attributes. Currently, only the camera dependency exists, which establishes a relationship between the camera (which is, strictly speaking, not an attribute) and a specific attribute. If a dependency is attached to an attribute, it influences this attribute based on the mutual distance between the main camera and the Particle System. The construction is:

attribute_name   #value
camera_dependency   attribute_name   {camera dependency properties}

camera_dependency identifies the attribute and its value definition. The attribute value is influenced by the camera distance.
camera_dependency is the keyword to define a camera dependency.
The camera dependency properties are handled in the paragraph 'Camera Dependency Properties'.

Simple Example: 

emission_rate                  30
camera_dependency              emission_rate
{
    distance_threshold         500
    increase                   false
}

Camera Dependency Properties


Increase

Determines whether the camera distance causes the attribute value to increase or decrease. See 'Distance Threshold' for further explanation. Syntax:

increase   true | false

increase is the keyword.
true | false defines whether the Attribute value is increased (true) or not. Default: true

Distance Threshold

The distance threshold is used to determine the speed of increasing/decreasing the attributes' value and defines when the effect no longer is applied. Syntax:

distance_threshold   #threshold

distance_threshold is the keyword.
#threshold defines the camera distance when the attribute is no longer influenced.

if 'increase' is true, the attribute is increased if the camera moves away from the technique. This effect works as long as 0 < camera distance < threshold. If the camera distance exceeds the threshold, the attribute value is no longer influenced.

if 'increase' is false, the attribute is decreased if the camera moves away from the technique. This effect works as long as camera distance >= threshold. If 0 < camera distance < threshold, the attribute value is no longer influenced.
