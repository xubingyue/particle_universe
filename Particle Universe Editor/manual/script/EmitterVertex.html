Vertex Emitter

Emits particles from the surface of a mesh. By traversing through the vertices, the spawn points are generated and particles are emitted from points that match the order of the vertices.

Definition

Syntax:

emitter   Vertex   [emitterName]   {emitter properties}

See ParticleEmitter for explanation of the syntax

Example: 

emitter                  Vertex
{
   mesh_name             athene.mesh
   step                  2
   number_of_segments    3
   number_of_iterations  2
   emission_rate         100
   ...
}

Properties


Vertex Step

Instead of traversing through all of the vertices, it is also possible to skip some vertices. The higher the vertex step, the more vertices are skipped. Syntax:

step   #value

step is the keyword. vertex_em_step is deprecated from version 1.2
#value is a short value > 0 that defines the step. Default: 1.

Number of Segments

The number of segments defines the number of generated spawn points on one edge. Syntax:

number_of_segments   #value

number_of_segments is the keyword. vertex_em_segments is deprecated from version 1.2
#value is a short value > 0 that defines in how many segments a edge will be divided. The number of spawnpoints is vertex_em_segments + 1. Default: 1.

Number of Iterations

In each particle technique update, a number of spawn points is generated (depends on number of segments). However, it is possible that the number of requested particles exceeds the number of existing spawn points. This means that the emitter has to wait until enough spawn points are generated (Note, that all spawn points are not precreated, but generated gradually). By increasing the number of iterations, the number of generated spawn points is also increased. Side-effect is, that increasing the number of iterations decreases the FPS. Syntax:

number_of_iterations   #value

number_of_iterations is the keyword. vertex_em_iterations is deprecated from version 1.2
#value is a short value > 0 that defines the number of iterations used to generate spawn points. Default: 1.

Mesh Name

Defines the name of the mesh of which the vertices are used. Syntax:

mesh_name   #name

mesh_name is the keyword. vertex_em_mesh_name is deprecated from version 1.2
#name is the name of the mesh.
