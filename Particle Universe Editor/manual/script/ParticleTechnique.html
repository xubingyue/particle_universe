Particle Technique

Definition

The syntax of a Particle Technique definition is:

technique   [techniqueName]   {technique properties}

technique is the keyword.
techniqueName is a an optional name to identify a technique. If you want to perform certain activities with this technique - emitting the technique for example - the name must be provided. There are no restrictions to the name.
The technique properties are handled in the paragraph 'Properties'.

Example: 

system                              exampleSystem
{
   technique                        exampleSystem::exampleTechnique
   {
       visual_particle_quota        1500
       material                     exampleMaterial
       use_alias                    myDefaultBillboardRenderer
       emitter
       {
          ...
       }
   }
}

Properties


Enabled

Enables or disables the technique as soon as the particle system starts. It can be convenient to disable the technique by default and enable it by means of another technique or through code. Syntax:

enabled   true | false

enabled is the keyword.
true | false defines whether the technique is enabled (true) or disabled (false). Default: true

Position

Position applies to the relative position of this technique in respect to the Particle System. Syntax:

position   #x #y #z

position is the keyword.
#x is a real value that defines the x coordinate of the techniques' position. Default: 0.
#y is a real value that defines the y coordinate of the techniques' position. Default: 0.
#z is a real value that defines the z coordinate of the techniques' position. Default: 0.

Keep Local

If this attribute is set to 'true', the particles are moving relative to the technique. Beware that this property can also be set in the particle system or the emitter, which makes it possible to keep emitted emitters, techniques, systems or visual particles local to respectively the particle system or the emitter. Syntax:

keep_local   true | false

keep_local is the keyword.
true | false if set to 'true', the particles are moving relative to the technique. Default: false.

Visual Particle Quota

The maximum number of visual particles that can be emitted. Syntax:

visual_particle_quota   #value

visual_particle_quota is the keyword.
#value is a real value > 0 that defines the maximum number of visual particles that can be emitted. Default: 500.

Emitted Emitter Quota

The maximum number of emitter particles that can be emitted. Syntax:

emitted_emitter_quota   #value

emitted_emitter_quota is the keyword.
#value is a real value > 0 that defines the maximum number of emitter particles that can be emitted. Default: 0.

Emitted Affector Quota

The maximum number of affector particles that can be emitted. Syntax:

emitted_affector_quota   #value

emitted_affector_quota is the keyword.
#value is a real value > 0 that defines the maximum number of affector particles that can be emitted. Default: 0.

Emitted Technique Quota

The maximum number of technique particles that can be emitted. Syntax:

emitted_technique_quota   #value

emitted_technique_quota is the keyword.
#value is a real value > 0 that defines the maximum number of technique particles that can be emitted. Default: 0.

Emitted System Quota

The maximum number of particle system particles that can be emitted. Syntax:

emitted_system_quota   #value

emitted_system_quota is the keyword.
#value is a real value > 0 that defines the maximum number of particle system particles that can be emitted. Default: 0.

Material

Set the material used in the renderer. Although the material is used in the renderer, it is defined in the technique. If the renderer is defined as an alias script (and an alias is assigned to it), it must be generic. Setting a material in the renderer is not considered 'generic'. Syntax:

material   #materialName

material is the keyword.
#materialName is the name of the material defined in a *.material script (or created by code of course). Default: "BaseWhite"

Lod index

Assign a level-of-detail (LOD) index to this Technique. The distance at which a LOD level is applied is determined by the lod_distances attribute of the containing Particle System. If lod_distances is not defined in the Particle System, lod_index has no meaning. Increasing indexes denote lower levels of detail. Note, that a lower level of detail can be accomplished by using a billboard renderer instead of a mesh renderer, but also the use of physics in near particles and not in particles that are further away. Syntax:

lod_index   #index

lod_index is the keyword.
#index is a real value > 0 that defines the index that corresponds with a certain LOD distance. Default: 0.

Default Particle Width

Set the default width of a particle. All particles in the Particle Technique can have either defaults dimensions, which means that all particles have the same width, height and depth, or own dimensions, which means that every particle has its own width, height and depth. Syntax:

default_particle_width   #width

default_particle_width is the keyword.
#width is a real value > 0 that defines the default width. Default: 20.

Note, that default particle width is an attribute that can be influenced by a camera dependency using the << camera_dependency construction.

Default Particle Height

Similar to default particle width. Syntax:

default_particle_height   #height

default_particle_height is the keyword.
#height is a real value > 0 that defines the height. Default: 20.

Note, that default particle height is an attribute that can be influenced by a camera dependency using the << camera_dependency construction.

Default Particle Depth

Similar to default particle width. Syntax:

default_particle_depth   #depth

default_particle_depth is the keyword.
#depth is a real value > 0 that defines the depth. Default: 20.

Note, that default particle depth is an attribute that can be influenced by a camera dependency using the << camera_dependency construction.

Spatial Hashing Cell Dimension

Spatial Hashing is a method that employs a hash a function for compressing a potentially infinite regular spatial grid. The Spatial Hashing method can be used in the Particle Technique in cases where the distance between particles is required. Examples are interparticle collision, flocking behaviour and avoidance. Activation of this algorithm is transparent and done by the affectors.
The 3d space is divided in grid cells with a certain dimension (size). The cell size is equal for x, y and z dimension. This attribute defines the cell dimension. Syntax:

spatial_hashing_cell_dimension   #value

spatial_hashing_cell_dimension is the keyword.
#value defines the x, y and z dimension of a cel. Default: 15

Spatial Hashing Cell Overlap

The cell overlap is used to put a particle in multiple cells if needed. This is a better way to determine nearby particles. A particle in one cell is not considered nearby in relation to a particle in another cel, although they can be nearby in terms of their world position. If you want to inspect all particles positioned in a radius around a central particle, and that particle is close to the border of a cel, you miss a few particles in the neighbouring cell that are nearby. By defining an overlap you duplicate particles in multiple cells, but you are able to determine all nearby particles. Syntax:

spatial_hashing_cell_overlap   #value

spatial_hashing_cell_overlap is the keyword.
#value is amount ov overlap between two cells. Default: 0.0 (no overlap)

Spatial Hashtable Size

The effectivity of the Spatial Hashing algorithm depends on the size of the hashtable. If the table becomes too large, particles clusters are spread out too much and groups of particles are no longer distinguished (if only one particle exists in a table row, collision detection is not executed for that row). If the table is too small, the whole idea of spatial clustering is lost. With a small table, a hashtable row may contain a lot of particles that all have to be validated with each other for collision. Defining the appropriate size of the table needs a little tuning. Use an appropriate combination of cell dimension, table size and update interval. Syntax:

spatial_hashtable_size   #value

spatial_hashtable_size is the keyword.
#value defines the size of the hashtable. Default: 50

Spatial Hashing Update Interval

When used, the (spatial) hashtable is by default updated every frame. This update interval offers the possibilty to update it after a specfied amount of time. Syntax:

spatial_hashing_update_interval   #value

spatial_hashing_update_interval is the keyword.
#value defines after which amount of time the hashtable is updated. Default: 0.05

Maximum Velocity

In some cases it is desirable to limit the velocity. If the particle system represents a flock of birds, it looks strange if the velocity of a bird becomes unnaturally high. This property limits the velocity to a maximum value. Syntax:

max_velocity   #value

max_velocity is the keyword.
#value is the maximum velocity of a particle. Default: Infinite.

Use alias

Use an alias. Renderers, Affectors, Emitters, Observers, Behaviours and Externs can all be defined in an alias script, which can be used in the technique. On the position of the 'use_alias', that piece of alias script is inserted. Syntax:

use_alias   aliasName

use_alias is the keyword.
aliasName is the name of an alias defined previously.
