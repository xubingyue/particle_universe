Setup

Particle Universe

Instal the Particle Universe package by starting the installer. Choose ../ogre as the target directory
This creates

  • ../ogre/PlugIns/ParticleUniverse, which includes the sourcecode and the documentation.
  • the ParticleUniverseViewer in ../ogre/Samples.
  • the AtlasImageTool in ../ogre/Tools.
  • the Particle Universe media in ../ogre/Samples/Media/ParticleUniverse.

Add the entry Plugin=ParticleUniverse to Plugins.cfg or Plugin=ParticleUniverse_d if you want to use it in Debug mode (see also the Plugins.cfg_ file in the Common directory).

Copy the content of the resources.cfg_ (in the Common directory) to resources.cfg. This contains the following entries:

  • FileSystem=../../../Media/ParticleUniverse
  • FileSystem=../../../Media/ParticleUniverse/core
  • FileSystem=../../../Media/ParticleUniverse/gui
  • FileSystem=../../../Media/ParticleUniverse/materials
  • FileSystem=../../../Media/ParticleUniverse/scripts
  • FileSystem=../../../Media/ParticleUniverse/models
  • FileSystem=../../../Media/ParticleUniverse/textures

Beware that the 'core' directory is mandatory for certain Particle Universe functions, so it may not be removed.
Optional you can add the directories of the Mediapack:

  • FileSystem=../../../Media/ParticleUniverse/mediapack/materials
  • FileSystem=../../../Media/ParticleUniverse/mediapack/scripts
  • FileSystem=../../../Media/ParticleUniverse/mediapack/models
  • FileSystem=../../../Media/ParticleUniverse/mediapack/textures

PhysX

If the PhysX engine is used in Particle Universe, you have to:
1. Uncomment the line #ifdef PU_PHYSICS in ParticleUniversePrerequisites.h (by default the PhysX engine is supported in Particle Universe, but other physics engines can also be added).
2. Set the environment variable PHYSX_HOME. For PhysX version 2.8.1. this can be done by the command:
set PHYSX_HOME=C:\Program Files\NVIDIA Corporation\NVIDIA PhysX SDK\v2.8.1\SDKs\
The PHYSX_HOME environment variable is used in the compiler settings.
3. Add PhysXLoader.lib to the linker settings (remark: Adding pragma comment ( lib, "PhysXLoader.lib") to the code doesnÂ´t work, because it cannot find the .lib even though the paths are set). Linking to PhysXLoader.lib can be omitted if the client application itself already links to this lib.
4. The project properties contains references to PhysX directories ($(PHYSX_HOME)Physics\include, for example). If you don't want to use Physx you can remove them or just keep them in. If you keep the directories, the compilation returns a Project : warning PRJ0018. This is harmless, so there is no urgent reason to remove these references.
