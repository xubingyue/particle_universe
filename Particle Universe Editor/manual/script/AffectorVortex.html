Vortex Affector

The Vortex Affector makes it possible to rotate particles around a given rotation vector with a given rotation speed.

Definition

Syntax:

affector   Vortex   [affectorName]   {affector properties}

See ParticleAffector for explanation of the syntax.

Example: 

affector                   Vortex
{
    rotation_axis          0 1 0
    rotation_speed         10
}

Properties


Rotation Axis

Defines the rotation vector. Syntax:

rotation_axis   #x   #y   #z

rotation_axis is the keyword. vortex_aff_vector is deprecated from version 1.2
#x is a real value that defines the x value of the rotation vector. Default: 0.0.
#x is a real value that defines the y value of the rotation vector. Default: 0.0.
#z is a real value that defines the z value of the rotation vector. Default: 0.0.

Rotation Speed

Defines the rotation speed. Syntax:

rotation_speed   #value |
   dyn_random {random properties} |
   dyn_curved_linear {control point [control point]} |
   dyn_curved_spline {control point [control point]} |
   dyn_oscillate {oscillating properties}

rotation_speed is the keyword. vortex_aff_speed is deprecated from version 1.2

#value is a real value that defines the rotation speed. Default: 1.0.

dyn_random {random properties} means that the rotation speed is generated randomly.

dyn_curved_linear {control point [control point]} means that the rotation speed is determined by means of 1 or more control points. The interpolation between the control points is linear. Note, that if this option is used, speed is based on the time since the start of the particle system.

dyn_curved_spline {control point [control point]} means that the rotation speed is determined by means of 1 or more control points. The interpolation between the control points is cubic. Note, that if this option is used, speed is based on the time since the start of the particle system.

dyn_oscillate {oscillating properties} means that the rotation speed is determined by means of an oscillating function.

See also the page 'Special Cases' for an explanation of the 'Dynamic Attribute'.
